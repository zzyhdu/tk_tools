# 智能打包计算器

基于 React + Vite + Tailwind CSS 的外箱尺寸推荐工具，支持：
- 根据单个商品尺寸和打包数量，计算最接近正方体的排列方案。
- 基于成本与目标利率，预测建议售价（含体积重/计费重参考）。

## 售价预测（当前版本）

可录入的成本项：
- 商品采购成本
- 快递成本（货源地到家）
- 快递成本（货物到国内头程商仓库）
- 头程到海外仓（自动计算）
- FBT 履约费

支持头程渠道选择：
- FBT-US特快海卡
- FBT-US标快海卡（以星合德带车架统配）
- FBT-US经济海卡（OA普船）
- FBT空派限时达

说明：
- 体积重按 `长 × 宽 × 高(cm) / 6000` 计算。
- 计费重按 `max(实际重量, 体积重)` 计算。
- 头程费用按“渠道报价（元/kg）×计费重量”自动计算，报价可随市场波动手工更新。
- `FBT空派限时达` 支持按目标海外仓分区（美西/美中/美东）+ 重量梯度（12-20、21-100、101+）自动计费，且按单件最低 12KG 起计。
- `FBT-US特快海卡` 支持按“目标海外仓代码 + 发货地（华东/华南/福建）+ 重量梯度（12/51/100/500/1000KG+）”自动计费；若某仓某档位无报价会提示无法计费。
- `FBT-US标快海卡` 支持按“目标海外仓代码 + 发货地（华东/华南/福建）+ 重量梯度（12/51/100/500/1000KG+）”自动计费；若某仓某档位无报价会提示无法计费。
- `FBT-US经济海卡` 支持按“目标海外仓代码 + 发货地（华东/华南/福建）+ 重量梯度（12/51/100/500/1000KG+）”自动计费；若某仓某档位无报价会提示无法计费。

## 发布到 Vercel

### 方式 1：网页导入（推荐）

1. 把当前项目推送到 GitHub/GitLab/Bitbucket。
2. 打开 Vercel，新建项目并选择该仓库。
3. Framework 选择 `Vite`（通常自动识别）。
4. 保持默认构建设置（仓库里已包含 `vercel.json`）。
5. 点击 Deploy。

### 方式 2：CLI

```bash
npm i -g vercel
vercel login
vercel
vercel --prod
```

首次执行 `vercel` 时按提示选择：
- Project Name: 自定义
- Framework: `Vite`
- Build Command: `npm run build`
- Output Directory: `dist`
